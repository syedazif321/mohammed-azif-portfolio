Dynamic Pick from Moving Conveyor

ROS 2 C++ Computer Vision ArUco Dynamic Grasping Control

Project Overview

This project tackles one of the most challenging problems in mobile manipulation: dynamically grasping a moving target from a moving platform. The simulation features an AMR (with a robotic arm) navigating alongside a conveyor belt. The core task was to successfully pick an ArUco-marked box from the moving conveyor while the AMR itself was in motion and not synchronized to the conveyor's speed.

Key Features & Technical Deep Dive

  • Real-Time 6-DoF Pose Estimation: I implemented a vision pipeline in C++ using ArUco markers to detect and calculate the full 6-DoF pose (position + orientation) of the moving box in real-time.
  • Relative Velocity Control: The system's "brain" was its ability to estimate the relative velocity between the robot's end-effector and the moving box. This was essential for synchronizing the arm's motion.
  • Dynamic Cartesian Tracking: The robotic arm operated in Cartesian space, continuously re-planning its path to track the box's real-time position. This control logic had to account for detection latency and ensure smooth, stable motion.
  • Precision Grasp Logic: A grasp was only triggered when the system's calculations determined the box was within a precise tolerance window for both position and velocity. This logic prevented failed attempts and ensured a successful pick, even with both platforms moving at different, varying speeds.
